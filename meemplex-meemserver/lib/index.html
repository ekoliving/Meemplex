Maji third-party JARs

This repository contains all of the third-party JARs required to build and run the Maji system. It also contains the maji meem and system jars which constitute the meem API.

Note: When committing new JARs or newer versions ...

  • Remove the version number from the JAR's name
  • Document the version number in the CVS log

Ant build tool - ant.jar

http://ant.apache.org

Ant build tool. Required for the launcher that ships with the installer.

Bouncy Castle provider - bcprov.jar

http://www.bouncycastle.org

Java JCE security providers. Used in Maji for creating certificates.

Bean Scripting Framework - bsf.jar

http://jakarta.apache.org/bsf

Apache Jakarta Bean Scripting Framework (BSF) used by Maji to allow interactive scripting by other languages. In particular, BeanShell.

BeanShell interactive Java scripting - bsh.jar version 2.0b1

http://www.beanshell.org

Provides an interactive Java scripting language that can be embedded within applications. Works with BSF.

Note: For BSH version 1.2b7. Since BSH has moved from IBM to Apache, the BeanShellBSFEngine source needed to be updated so that it uses "org.apache.bsf.*" packages, not the old "com.ibm.bsf.*" packages. This bsh.jar has been updated accordingly.

Jini Davis JARs - davis/*.jar

http://davis.jini.org

Jini Davis project libraries. These are replacement libraries for the Jini SDK that provide RMI security features.

Eclipse IDE JARs - eclipse/*.jar

http://www.eclipse.org

Eclipse Integrated Developer Environment (IDE) for Java.

GNU command line parser - getopt.jar

http://www.gnu.org/software/java/java-software.html

Command line parser (getopt) used by the Maji Launch for stand-alone JVMs.

Install Anyware classes - IAClasses.zip

Pinched from the enterprise installed version of Install Anywhere

This is only needed to build the majitek-install project.

JavaBeans Activation Framework- activation.jar

http://java.sun.com/beans/glasgow/jaf.html

Required by JavaMail (and thus MeeMail). JavaBeans Activation Framework provides services to determine the type of an arbitrary piece of data, encapsulate access to it, discover the operations available on it, and to instantiate the appropriate bean to perform said operation(s).

Java Development Tools - jdtcore.jar

http://www.eclipse.org

Provides bytecode parser for classlibrary. This jar comes from the org.eclipse.jdt.core_2.1.0 eclipse plugin.

JavaMail - mail.jar

http://java.sun.com/products/javamail/

Required by MeeMail. The JavaMail 1.3.1 API provides a set of abstract classes that model a mail system. The API provides a platform independent and protocol independent framework to build Java technology-based mail and messaging applications.

Java Serialization to XML - jsx.jar

http://www.csse.monash.edu.au/~bren/JSX

Provides Java object importing and exporting in XML format. Used by MeemSpace to provide MemeDefinition and MeemContent persistence.

This JAR must be licensed for commercial use - WHICH WE HAVE NOT DONE YET. We'll have to decide whether to purchase or replace this JARS.

JUnit (unit testing) - junit.jar

http://www.junit.org

Provides command line and GUI unit testing tools.

Maji meem and system API - maji/maji-meem.jar, maji/maji-system.jar

http://www.majitek.com

The Maji meem API jars - not 3rd party.

Software Zoo library - swzoo.jar

http://www.swzoo.org

Contains general utilities and sub-systems. such as the Logging infrastructure.

Apache XML-RPC - xmlrpc.jar

http://ws.apache.org/xmlrpc

Java implementation of XML-RPC , a protocol that uses XML over HTTP to implement remote procedure calls.

Jetty Web Server and Servlet Container - jetty/javax.servlet.jar, jetty/org.mortbay.jetty.jar

http://jetty.mortbay.org/jetty/

Jetty is a 100% Java HTTP Server and Servlet Container, that can be easily included in another application.

Java Service Wrapper - a launcher/wrapper for Java services

http://wrapper.tanukisoftware.org/

JSW is a multi-platform wrapper for Java services. It provides start, stop, restart and automatic restart.
